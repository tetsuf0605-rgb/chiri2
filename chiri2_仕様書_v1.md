# chiri2（世界地理ドリル）仕様書 v1

作成: 2026-07 ／ このチャットで合意 → Claude Code で実装

---

## 1. 概要

- 世界地理の白地図クイズ＋（将来）rekishi型クイズのハイブリッドアプリ。
- 第1弾: **東南アジア**（地名ドリル②の範囲）。以降、写真9枚の範囲（ヨーロッパ／ロシア／南アジア／西アジア／アフリカ／北アメリカ／南アメリカ／オセアニア）を順次追加。
- リポジトリ: `tetsuf0605-rgb/chiri2`（main）／ 公開URL: `https://tetsuf0605-rgb.github.io/chiri2/`
- 単一 `index.html`・vanilla JS・外部依存なし・PWA 3点セット（icon.png / manifest.webmanifest / headタグ）・`.nojekyll` を初回に追加。
- localStorage キー: `chiri2_v1`（成績・にがて復習・中断再開・記録リセット）。
- 手書き回答: **なし**（選択・タップ・記述＋自己採点のみ）。

## 2. デザイン（iOS純正風）

target系で確立したスタイルを踏襲する。

- Appleシステムフォント（-apple-system）
- Large Title ナビゲーションバー（スクロールで縮小）
- Inset Grouped リスト、シートモーダル、ブラー付きタブバー
- セーフエリア対応（env(safe-area-inset-*)）
- ライト基調。アクセントカラーは地図アプリらしい青系（#007AFF）を仮とする。

## 3. 画面構成

タブバーで2モードを切り替える。

1. **地図モード**（今回実装）
   - 地域選択（Inset Groupedリスト。まず「東南アジア」のみ）
   - 出題方向A「位置あて」: 地名を提示 → 地図上の該当パーツをタップ
   - 出題方向B「名前あて」: 地図上のパーツをハイライト → 4択 or 記述（自己採点フォールバック）
   - 正解時はパーツを点滅ハイライト、誤答時は正解位置を表示
2. **クイズモード**（枠だけ実装・データ空）
   - rekishi型エンジン（TERMS / POOL / FIGS）を同梱するが、`TERMS` が空の間はメニューに「準備中」表示。
   - ノートができたらデータ行を追加するだけで有効化。**本体ロジックは触らない。**

## 4. データ構造

### 4.1 MAPS（新設・地図モード用）

```js
const MAPS = [
  {
    id: "sea",                 // 地域ID
    title: "東南アジア",
    viewBox: "0 0 700 560",
    // SVGは生成スクリプトが出力したものを焼き込む
    // 各出題パーツは data-part 属性で番号を持つ
    parts: [
      // kind: 国 / 島 / 半島 / 海峡 / 海 / 川 / 湖 / 山脈 / 台地 / 都市 / 遺跡 など
      { no: 1, name: "マレー半島", yomi: "まれーはんとう", alt: [], kind: "半島", hit: "area" },
      { no: 2, name: "マラッカ海峡", yomi: "まらっかかいきょう", alt: [], kind: "海峡", hit: "zone" },
      // ...
    ]
  }
];
```

- `hit`: タップ判定の型。`area`=ポリゴン自体、`zone`=海峡・岬・都市など点/線対象用の透明ヒット円（半径はiPhoneで44pt相当を確保）。
- **安全装置**: 出題対象は「SVG内に実際に `data-part` 付きで描かれた番号」だけを対象にする（rika2の教訓を踏襲）。parts配列にあっても描画がなければ出題しない。
- 選択肢の質: ダミーは同じ `kind` から選ぶ（海峡には海峡、島には島）。

### 4.2 TERMS / POOL / FIGS / CHAPTERS（rekishi型・今回は空）

- rekishi/rika2 と同一スキーマ。`CHAPTERS` は地域（東南アジア＝0…）と対応させる。
- 採点は共通ルール§5どおり: 表記ゆれ吸収（空白・かなカナ・全半角）、よみ一致OK、自動採点→自己採点フォールバック。

## 5. 白地図SVGの生成（方式A）

- データ源: **Natural Earth 50m**（パブリックドメイン）
  - `ne_50m_admin_0_countries`（国境）
  - `ne_50m_rivers_lake_centerlines`（河川）
  - 必要に応じ `ne_50m_lakes`
- 生成場所: **Claude Code（てつやのPC）で実行する Python スクリプト `tools/make_map.py`**
  - このチャットがスクリプトを作成 → Claude Code が実行 → 出力SVGを `index.html` に組み込む。
  - 依存は標準ライブラリ＋urllib のみを目標（GeoJSON配布物を使用）。
- 処理内容:
  1. 指定bboxで切り抜き（東南アジア: 東経90°〜145°、南緯12°〜北緯28°を仮とする）
  2. 正距円筒図法で投影 → viewBox座標へ
  3. 線の簡略化（タップしやすさとファイルサイズの両立。目標: 地域SVGあたり60KB以下）
  4. 出題パーツに `data-part="番号"` を付与、海峡等には透明ヒット円を配置
- 生成物はビルド時のみ使用。**アプリ実行時は完全オフラインで動く**（SVGは焼き込み済み）。

## 6. 東南アジア 仮候補リスト（ドリル②紙面より・要確認）

リスト確定まで、紙面のラベル種別から以下を候補とする。**答えの対応（ひらがな→実名）はてつやのリスト到着後に確定。**

- 半島: インドシナ半島・マレー半島 ほか
- 海峡: マラッカ海峡・ロンボク海峡 ほか（け海峡・こ海溝などは紙面対応を確認）
- 島: スマトラ・ジャワ・カリマンタン（ボルネオ）・スラウェシ・ルソン・ミンダナオ・バリ・ティモール ほか、資源タグ付き（a: ボーキサイト、b/c: すず）
- 海・海溝: 南シナ海・アンダマン海・ジャワ海・バンダ海・フィリピン海溝 ほか
- 川: エーヤワディー川・チャオプラヤ川・メコン川・ホン川 ほか
- その他: アンコールワット・スプラトリー諸島・クラ地峡・コラート台地
- 都市: 1〜20（各国首都・主要都市。番号対応はリスト待ち）

## 7. ワークフロー（地域追加の標準手順）

1. てつやが地図写真＋出題地名リストをこのチャットへ
2. チャットが `parts` 対応表＋`make_map.py` の地域設定を作成
3. Claude Code がスクリプト実行 → SVG生成 → `index.html` へ組み込み（Keepを押す）
4. ターミナルで git add / commit / push → GitHub Pages 反映確認
